# User Flows

## New Client Lead to Project

1. Yaniv creates or receives a client lead.
2. Client details and initial notes are stored.
3. Discovery questions are captured.
4. Yaniv creates a project record.
5. AI can help summarize the need into a draft brief.
6. Yaniv reviews and edits the brief.
7. The project moves toward pricing and scope.

## Discovery to Scope

1. Discovery notes, files, links, and decisions are attached to the project.
2. Yaniv defines goals, assumptions, constraints, and exclusions.
3. A draft scope is created.
4. Internal supplier-facing notes can be separate from client-facing scope.
5. Yaniv reviews the scope before sharing it with the client.

## Pricing and Approval

1. Yaniv estimates supplier cost.
2. Yaniv sets client price.
3. The system calculates margin.
4. Yaniv decides whether the margin is acceptable.
5. Client receives the scope and price for approval.
6. Client approves, rejects, or requests changes.

## Payment or Paid Hours Gate

1. Approved project waits for payment or available hour bank.
2. Work remains blocked until one of these is true:
   - Required payment is marked as received.
   - Enough paid hours are available in the client's hour bank.
3. Once ready, Yaniv can assign supplier work.

## Supplier Assignment

1. Yaniv selects an approved supplier.
2. Supplier sees only assigned work, relevant files, instructions, and expected time rules.
3. Supplier does not see client price, agency margin, or unrelated client context.
4. Supplier submits updates and time entries.

## Client Change Request

1. Client submits a requested change.
2. Request is marked as needing agency review.
3. Yaniv reviews impact on scope, cost, supplier time, and timing.
4. Yaniv prices the change.
5. Client approves or declines the priced change.
6. Only approved changes become supplier work.

## Supplier Time Review

1. Supplier logs time against assigned project work.
2. Yaniv reviews submitted entries.
3. Approved time updates supplier amount owed.
4. Client hour bank usage may update if the project is hour-based.
