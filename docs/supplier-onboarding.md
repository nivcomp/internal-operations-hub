# Supplier Onboarding

Supplier onboarding should collect enough information for Yaniv to decide whether and how to use a supplier.

## Supplier Registration Fields

- Name
- Email
- Phone
- Country
- Timezone
- Languages
- Main skills
- Tools
- Years of experience
- Hourly rate
- Weekly availability
- Portfolio links
- What else do you know that is not listed?

## Skill Suggestions

The "What else do you know that is not listed?" field should create supplier skill suggestions.

These suggestions should require agency approval before becoming official catalog skills.

Example:

- Supplier writes: "Webflow audits, CRM cleanup, Make automation"
- System creates three pending suggestions.
- Yaniv can approve, reject, or edit them.

## Supplier Statuses

- Pending review
- Approved
- Inactive

## Onboarding Rules

- Supplier cannot see client-sensitive information during onboarding.
- Supplier cannot self-approve skills.
- Supplier hourly rate is internal to agency operations.
- Supplier profile should help Yaniv match skills, availability, and cost to project needs.
