# Product Vision

Client-to-Scope AI is an internal operating system for Yaniv's client work. It helps turn loose client conversations into structured projects with clear scope, pricing, approvals, paid hours, supplier assignments, time tracking, change requests, and organized communication.

The product should reduce the operational drag that happens between a client asking for help and a project becoming profitable, approved, and deliverable.

## Primary User

The first user is Yaniv.

This is not initially a self-serve SaaS product for agencies. It should be built around one real operator managing real clients, suppliers, costs, scope decisions, and delivery risk.

## Core Jobs

- Capture new client needs in one place.
- Turn discovery notes into structured briefs.
- Define scope before work begins.
- Keep client-facing commitments separate from supplier-facing instructions.
- Set client price and supplier cost separately.
- Track paid hours and payment status before work starts.
- Assign suppliers only when the project is ready.
- Track supplier time and amount owed.
- Manage client approval and change requests.
- Keep files, links, notes, and decisions attached to the correct project.
- Use AI to assist communication and structure, without giving AI final authority.

## Product Shape

The system should feel like a practical internal dashboard:

- Calm, simple, and fast to scan.
- Focused on active projects and decisions.
- Clear about what is blocked, waiting, approved, paid, or ready.
- Built around records and workflows rather than marketing screens.

## Long-Term Direction

If the internal version works well, it could later become a repeatable agency operating system. That should not drive the first build. The first build should solve Yaniv's own workflow cleanly.
