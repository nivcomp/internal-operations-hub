# Internal Use MVP

The MVP should help Yaniv run client projects with less manual tracking and fewer unclear handoffs.

## MVP Goals

- See all clients, projects, suppliers, approvals, payments, and change requests from one internal dashboard.
- Create a project from a client lead or existing client.
- Capture discovery notes and convert them into a project brief.
- Create a scope that can be reviewed and approved by the client.
- Store client price, supplier cost, and expected margin separately.
- Track whether payment has been received or paid hours are available.
- Assign supplier work only when the start rules are satisfied.
- Let suppliers log time against assigned work.
- Let Yaniv review supplier time and know what is owed.
- Let clients request changes without those requests automatically becoming work.

## MVP Non-Goals

- Public SaaS onboarding
- Subscription billing
- Marketplace supplier discovery
- Automated AI project management
- Complex permissions
- Advanced analytics
- Deep accounting integration
- Native chat replacement
- Full document signing

## First Useful Screens

- Internal dashboard
- Client list and client detail
- Project list and project detail
- Supplier list and supplier detail
- Pricing and margin view
- Approval and payment status view
- Supplier time entries
- Change request queue
- Files and links section per project

## Definition of Done for MVP

The MVP is useful when Yaniv can answer these questions quickly:

- What projects are active?
- What is waiting on the client?
- What is waiting on me?
- Has the client approved the scope?
- Has the client paid or do they have hours available?
- Who is assigned?
- What has the supplier done?
- How much do I owe the supplier?
- What changes have been requested?
- Which requests need pricing before work begins?
